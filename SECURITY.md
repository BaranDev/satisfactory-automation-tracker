# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do not** open a public GitHub Issue for security vulnerabilities
2. Instead, use [GitHub's private vulnerability reporting](https://github.com/BaranDev/satisfactory-automation-tracker/security/advisories/new)

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Assessment**: Within 1 week
- **Fix**: Dependent on severity

## Scope

This policy applies to the latest version of the Satisfactory Automation Tracker and its dependencies.
